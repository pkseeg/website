Introduction¶

In this project we use two different decision tree models, one using a GINI optimization method, one using an entropy optimization method. The goal is to predict whether a breast tumor is benign or malignant, given 10 features present in an image of the mass.

Data for this project comes from the kaggle dataset found at https://www.kaggle.com/uciml/breast-cancer-wisconsin-data. From the dataset author: "Features are computed from a digitized image of a fine needle aspirate (FNA) of a breast mass. They describe characteristics of the cell nuclei present in the image." From each observation 10 features were extracted, as both mean measures and worst-case measures.

  • mean radius
  • texture (sd of gray scales)
  • perimeter
  • area
  • smoothness (variation in radius lengths)
  • compactness (perimeter^2 / area - 1.0)
  • concavity
  • concave points
  • symmetry
  • fractal dimension

Analysis¶

Decision Tree using GINI method¶

We'll start by using the default tree classifier from sklearn. The default tree uses the gini method for determining the best split at each level of the tree. The gini method minimizes the weighted average of (1-(sum of squared possibilites)) for each remaining attribute in the tree.

Here we'll create the tree, train the tree, and predict the responses for the test dataset.

In [6]:
# This creates the tree
basic_tree = DecisionTreeClassifier()
# This trains the tree
basic_tree = basic_tree.fit(x_train,y_train)

# This predicts the response for the test dataset
y_pred = basic_tree.predict(x_test)

Now we can use the metrics library from scikitlearn to determine the accuracy, precision and recall of our model on the test data. For our purposes we say that a 'positive' is a malignant tumor, and benign is 'negative.'

In [8]:
print("Accuracy:",metrics.accuracy_score(y_test,y_pred))
print("Precision:",metrics.precision_score(y_test, y_pred,pos_label='M'))
print("Recall:",metrics.recall_score(y_test, y_pred,pos_label='M'))

Accuracy: 0.9473684210526315

That's pretty good! Although decision tree models are often prone to overfitting, so we would likely want to use an ensembling technique, or perhaps a random forest model, before using this model on new data.

To display the tree, we can use the graphviz method from scikitlearn's tree library (and a couple other things) to define a function to let us see the tree.

In [9]:
from sklearn.tree import export_graphviz
from sklearn.externals.six import StringIO
from IPython.display import Image
import pydotplus

# Display the decision tree
def displayTree(tree):
    dot_data = StringIO()
    export_graphviz(tree, out_file=dot_data,  
                filled=True, rounded=True, impurity=True,
                special_characters=True,feature_names = features,class_names=['0','1'])
    graph = pydotplus.graph_from_dot_data(dot_data.getvalue())  
    graph.write_png('breast-cancer-tree.png')
    Image(graph.create_png())

Decision Tree using Entropy method¶

Entropy measures information gain, so the goal of this method is to maximize entropy at each level.

Here we build the new model, train it, predict on the test data, and measure the model's accuracy.

In [11]:
entropy_tree = DecisionTreeClassifier(criterion="entropy")

# Train entropy tree
entropy_tree = entropy_tree.fit(x_train,y_train)

# Predict response
y_pred_entropy = entropy_tree.predict(x_test)

# Accuracy
print("Accuracy:",metrics.accuracy_score(y_test, y_pred_entropy))
print("Precision:",metrics.precision_score(y_test, y_pred_entropy,pos_label='M'))
print("Recall:",metrics.recall_score(y_test, y_pred_entropy,pos_label='M'))

Accuracy: 0.9181286549707602
Precision: 0.8888888888888888
Recall: 0.8888888888888888

Entropy achieved slightly lower accuracy, precision, and recall measures than with the gini method, though still respectable. We can view this tree calling the function we defined earlier.

Again we see that the worst case perimeter was the most informative feature from the breast mass image. Here we saw that the gini method for selecting branching features in a decision tree classifier was slightly more effective than the entropy method on the same data. In further research a random forest classifier could be leveraged to obtain more accurate results.
